# A 股每日行情监控

## 项目目的

每个交易日收盘后（约 15:30 之后）生成一份简短中文行情简报，覆盖：

- 上证指数、深证成指、创业板指、沪深 300、科创 50 的收盘数据
- 当日涨跌幅、成交额
- 一段中性、客观的简短点评（不构成投资建议）

## 使用流程

### 1. 检查交易日

判断今天是否为 A 股交易日：周一至周五，且非中国法定节假日。如果不是，停止执行，输出 `今日非交易日，已跳过`。

### 2. 拉取行情数据

执行：

```bash
python scripts/fetch_market.py
```

脚本输出 JSON 到 stdout，结构如下：

```json
{
  "date": "2026-04-29",
  "source": "akshare",
  "indices": [
    {"code": "sh000001", "name": "上证指数", "close": 3000.00, "pct_change": 0.50, "volume_yi": 4200.0}
  ],
  "errors": []
}
```

如果 `indices` 为空数组，说明所有数据源都失败，参考下方"错误处理"。

### 3. 生成报告

按 `templates/daily_report.md` 的结构填充数据。占位符规则：

- `{date}`、`{source}` 直接替换
- 表格行按 `indices` 数组的顺序生成
- `{commentary}` 由你（Claude）基于数据自由撰写，**不超过 150 字**，要求：
  - 中文、客观中性
  - 描述行情特征（涨跌、量能、风格）
  - **不出现具体股票/板块买卖建议**
  - **不预测明日走势的方向性结论**

### 4. 输出与分发

将完整报告写入 `out/report_YYYYMMDD.md`（YYYYMMDD 用今天的日期）。

如果配置了 Slack 连接器，将报告内容（纯文本格式）发送到指定频道；如果配置了 Gmail 连接器，发送到指定邮箱。如果都没配置，仅写入文件即可。

## 错误处理

| 情况 | 处理方式 |
|---|---|
| 所有数据源失败 | 报告中标注"今日数据源全部不可用"，列出错误信息，不要编造数据 |
| 部分指数缺失 | 跳过该指数，其他正常输出，在简评中说明 |
| 节假日误触发 | 第 1 步直接退出 |
| 脚本超时（>60s） | 视作数据源失败，按"全部不可用"处理 |

## 核心约束

1. **数据真实性**：所有数字必须来自脚本输出，不允许任何形式的估计、补全或编造。
2. **不给投资建议**：简评保持描述性，避免"建议买入"、"应当减仓"、"看多/看空"等定向表述。
3. **中文输出**。
4. **输出长度可控**：完整报告控制在 500 字以内，便于在 Slack 或邮件中阅读。
